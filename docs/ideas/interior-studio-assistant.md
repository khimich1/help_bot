# Interior Studio Assistant — Telegram-бот для студии дизайна

## Проблема (How Might We)

Как мы можем дать студии из двух дизайнеров Telegram-бота, который на ходу (голосом) фиксирует задачи по общим проектам, напоминает о дедлайнах и собирает структурированный отчёт для клиента — без ручной сборки из заметок и папок?

## Рекомендуемое направление

**«Studio ReAct + Voice»** — ReAct-агент на LangGraph (паттерн `airline_react_agent.py`) + Telegram-слой на `python-telegram-bot` + SQLite на одном сервере.

### Архитектура (3 слоя)

1. **Telegram handler** — текст, голос (`voice.ogg` → OpenAI Whisper → текст), whitelist из 2 `telegram_user_id`.
2. **ReAct-агент** — `MessagesState`, `agent_node` ↔ `ToolNode`, conditional edge `tools` / `END`.
3. **Сервисы** — задачи, проекты, напоминания (APScheduler / job queue), отчёты (срез 2), RAG (срез 3).

### Голосовой pipeline (вне графа)

```
voice.ogg → Whisper → HumanMessage → agent.invoke()
```

### Контекст проекта

- `active_project_id` per user — tool `get_active_project` / `set_active_project`.
- Бот угадывает проект из контекста; уточняет только если совсем неясно.
- Пользователь может поправить задачу потом (низкий порог входа).

### Tools MVP

| Tool | Назначение |
|------|------------|
| `list_projects` | Список активных общих проектов |
| `create_project` | Новый проект (существующие + новые) |
| `get_active_project` | Текущий проект пользователя |
| `set_active_project` | «Работаем по Ивановым» |
| `create_tasks` | Одна или несколько задач (batch из голоса) |
| `list_tasks` | Фильтр: проект, мои/все, статус, просроченные |
| `complete_task` | Отметить выполненным |

### Tools срез 2 (отчёт)

| Tool | Назначение |
|------|------------|
| `generate_client_report` | 5 блоков за период (неделя / с последнего отчёта) |

### Напоминания

- **09:00 МСК** — личный дайджест каждому дизайнеру: просроченные + задачи на сегодня/неделю.
- **За 1 день до дедлайна** — разовое напоминание.
- Общий дайджест студии не нужен; по запросу — «мои / по проекту».

### Отчёт клиенту (срез 2)

Структурированный черновик в Telegram, дизайнер правит и отправляет клиенту сам:

1. Выполнено
2. В работе
3. Нужно от клиента
4. Следующие шаги
5. Риски / задержки

## Допущения для проверки

- [ ] Whisper + русский на объекте → ≥80% задач без правки текста (10 реальных голосовых)
- [ ] LLM стабильно вызывает `create_tasks`, а не «я бы создала…» (pytest, 10 типичных фраз)
- [ ] Двое пользуются ботом 2 недели, не возвращаются в заметки (пилот)
- [ ] Активный проект угадывается из контекста при 5–15 проектах

## MVP

**В scope (срез 1):**

- Telegram-бот, 2 пользователя (whitelist), общие проекты
- Проекты: ручное создание в боте (существующие заносятся при старте)
- Текст + **голос с первого дня**
- `create_tasks` → `list_tasks` → `complete_task`
- Контекст проекта (`set_active_project` / авто)
- Напоминания: дайджест 09:00 МСК + за день до дедлайна

**Happy path:**

> Маша на объекте шлёт голос: «По Ивановым — заказать плитку, Петя уточнит срок у поставщика, до пятницы скинуть варианты света»
> → Whisper → агент → `create_tasks` → «Создала 3 задачи по проекту Ивановы»

**Срез 2:** `generate_client_report` (5 блоков)

**Срез 3:** RAG «Справочник» — см. [`interior-studio-knowledge.md`](interior-studio-knowledge.md); Яндекс.Диск API — срез 3.1

## Not Doing (и почему)

| Фича | Причина |
|------|---------|
| Яндекс.Диск sync / RAG | MVP без Диска; доступ к API уточним позже |
| Google Calendar | Дедлайны в задачах + TG-напоминания достаточно |
| Клиенты в боте | Внутренний инструмент студии |
| PDF-отчёты | Текстовые блоки в чате |
| Мульти-агент | Overkill для 2 человек |
| Human gate (кнопки подтверждения) | «Создавай сразу, поправим потом»; v1.1 опционально |
| Импорт проектов из CSV/Excel | Ручной ввод 10–15 названий при старте |
| 15+ tools | Раздувает prompt, растут ошибки tool calling |
