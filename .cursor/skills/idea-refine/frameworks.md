# Линзы для генерации вариантов агентов

Выбирай 2–3 подходящие линзы, не прогоняй все механически.

## Базовые (из оригинала)

| Линза | Вопрос | Пример для агента |
|-------|--------|-------------------|
| Инверсия | Что если наоборот? | Только FAQ без действий; или только tools без свободного чата |
| Снятие ограничений | Без бюджета/времени? | Идеальный агент с human-in-the-loop на каждом шаге |
| Смена аудитории | Для кого ещё? | Агент для оператора, не для клиента |
| Комбинация | Слить с …? | Агент + уведомления в Telegram |
| Упрощение | В 10 раз проще? | 1 tool, 1 сценарий, без памяти |
| 10x | Масштаб? | Fleet агентов по доменам |
| Эксперт | Что очевидно инженеру? | Structured output вместо свободного текста |

## Специфичные для LangGraph

| Линза | Вопрос |
|-------|--------|
| **Tool-first** | Какие 3 API-операции покрывают 80% запросов? |
| **Graph-minimal** | Можно ли без conditional edges — линейный ReAct? |
| **Human gate** | Где нужен interrupt (оплата, отмена, удаление)? |
| **Mock-first** | MVP на `FLIGHTS_DB`-подобном mock без внешних API? |
| **Bilingual** | RU+EN в одном агенте или два промпта? |
| **Fallback** | Что если LLM не вызвал tool — эскалация на человека? |

## Домены (примеры направлений)

- **Support:** FAQ search → create ticket → status (3 tools)
- **Booking:** search → book → modify (как airline)
- **Internal:** query docs → run approved script (осторожно с security)
- **Copilot в IDE:** только read tools, без write в prod
