# Критерии оценки идей агентов

Рубрика для фазы 2 (Evaluate & Converge).

## 1. Ценность для пользователя

**Боль vs витамин:**
- **Боль:** пользователь тратит время/деньги сейчас; готов менять поведение
- **Витамин:** «прикольно», но завтра забудут

**Вопросы:**
- Назови 3 конкретных пользователя с этой проблемой
- Что делают **сейчас**? (конкурент — workaround, не ChatGPT)
- Как часто проблема? (ежедневно > раз в месяц)
- Pull (просят) или push (ты думаешь, что надо)?

**Красные флаги:**
- «Всем пригодится» — нет конкретного пользователя
- «Как ChatGPT, но для X» без tools/данных — маргинальная ценность

## 2. Реализуемость (LangGraph)

**Техника:**
- Достаточно ReAct + N tools или нужен RAG/мульти-агент?
- Есть ли источник данных (API, БД, mock как `FLIGHTS_DB`)?
- Самое сложное: prompt reliability? tool design? latency?

**Ресурсы:**
- MVP за дни (1 файл, 2–3 tools) или недели?
- Нужна ли персистентность (checkpointer)?

**Красные флаги:**
- «Сначала построим RAG на 10M документов»
- 15 tools в MVP
- Агент без верифицируемых tool results (галлюцинации)

## 3. Дифференциация

**Почему не просто чат с GPT?**
1. Доступ к **действиям** (tools) — сильнейший аргумент
2. Доменные данные (брони, тикеты)
3. Guardrails и политики
4. Русский UX + доменная логика (`resolve_date`)
5. Только «красивый prompt» — слабое отличие

## Аудит допущений

| Тип | Пример для агента |
|-----|-------------------|
| **Must be true** | LLM стабильно вызывает нужный tool при типичных запросах |
| **Should be true** | Пользователи пишут текстом, не кнопками |
| **Might be true** | Нужна история диалога > 10 сообщений |

## Матрица решений

| | Высокая реализуемость | Низкая |
|--|----------------------|--------|
| **Высокая ценность** | Делай первым | Риск оправдан |
| **Низкая ценность** | Только если тривиально | Не делай |

## MVP агента

1. **Одна job-to-be-done** — один happy path
2. **Рискованное допущение первым** — например, tool calling на реальных фразах
3. **Time-box** — «что за 2–3 дня», не список фич
4. **Not Doing обязателен**
5. **Стыдно показать** — если MVP кажется «готовым продуктом», переусложнил
