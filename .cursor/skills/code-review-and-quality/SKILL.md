# Ревью кода и качество

Пять осей: корректность, читаемость, архитектура, безопасность, производительность.

## Quality gates

Перед «готово»:
- [ ] Тесты проходят: `pytest -v`
- [ ] Агент запускается без ошибок
- [ ] Нет секретов в diff
- [ ] Русский UX сохранён (если применимо)
- [ ] Лимиты итераций / guardrails на месте

## Быстрый чеклист агента

- [ ] Tools атомарны, с docstring
- [ ] State минимален
- [ ] Граф простой, есть END
- [ ] `.env` для MODEL и API key
- [ ] Ошибки tools возвращаются LLM читаемо
- [ ] Нет hardcoded API keys

## Формат отчёта

См. персону `code-reviewer` — отчёт на русском.

## Когда эскалировать

- Security concerns → `security-auditor`
- Пробелы в тестах → `test-engineer`
- Архитектура раздута → упрости, skill `incremental-implementation`

## Правила

1. Не approve с Critical issues
2. Конкретные file:line в замечаниях
3. Отметь, что сделано хорошо
