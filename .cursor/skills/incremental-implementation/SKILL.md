# Инкрементальная реализация

Один срез → тест → верификация → коммит (если просят) → следующий срез.

## Цикл

```
Реализуй → Тест → Верифицируй → (коммит) → следующий срез
```

## Стратегии нарезки для агентов

### Вертикальный срез (предпочтительно)
```
Срез 1: Tool search_flights + unit-тест
Срез 2: Подключить tool к графу
Срез 3: System prompt + русские ответы
Срез 4: Guardrails (max iterations)
```

### Горизонтальный (избегать)
Все tools сразу, потом весь граф — риск большого нерабочего diff.

## Правила

1. Не больше ~100 строк без теста/запуска
2. Каждый срез — рабочее состояние
3. Следуй паттернам `airline_react_agent.py`
4. Объясняй пользователю на русском, что сделано в срезе
5. После среза — `test-driven-development` для логики

## Признаки, что срез слишком большой

- Больше 3 файлов без промежуточной проверки
- Нельзя запустить/протестировать промежуточный результат
- Смешаны refactor и новая фича
