# Interview Me

«Сделай агента» ≠ понятная задача. Этот skill — **до** `idea-refine` и `spec-driven-development`.

## Когда применять

- Нет **кто** / **зачем** / **успех** / **ограничения**
- Шаблонный запрос: «как у ChatGPT, но для …»
- Ты собираешься угадывать требования
- Пользователь: «поговори со мной», «уточни», «мы точно это хотим?»

**Не применять:** rename, typo, «как работает этот код», явная спека.

## Процесс

### Шаг 1: Гипотеза + confidence

```
ГИПОТЕЗА: Ты хочешь …
УВЕРЕННОСТЬ: ~40% — неясно: …
```

### Шаг 2: Один вопрос + guess

```
В: <один вопрос>
ДОГАДКА: <твоя версия ответа и почему>
```

Жди ответа. Не пачкой из 10 вопросов.

### Шаг 3: Лови «хочу» vs «должен хотеть»

Пользователь просит dashboard — может, нужен один отчёт в Telegram.  
Просит 10 tools — может, хватит 2.

### Стоп (~95%)

Можешь предсказать ответы на следующие 2–3 вопроса → выдай **подтверждённый intent**:

```markdown
## Подтверждённый intent
- Пользователь агента: …
- Job-to-be-done: …
- Успех: …
- Ограничения: …
- Не делаем: …
```

## Handoff

| Intent | Следующий skill |
|--------|-----------------|
| Идея есть, варианты нужны | `idea-refine` |
| Направление ясно | `spec-driven-development` |
| Архитектура | `agent-architect` |

## Тон

На русском. Прямой, без анкеты на 20 пунктов. Готов ошибиться в догадке — так быстрее.

## Верификация

- [ ] Confidence обоснован числом
- [ ] Вопросы по одному
- [ ] Intent зафиксирован до кода
- [ ] Handoff явный
