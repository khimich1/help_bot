# Планирование и декомпозиция

Разбей работу на маленькие верифицируемые задачи. Каждая — одна сессия фокуса.

## Процесс

### 1. Режим планирования (read-only)
- Прочитай спеку и код
- Найди паттерны (`airline_react_agent.py`)
- Построй граф зависимостей
- **Не пиши код**

### 2. Вертикальные срезы (предпочтительно)

**Плохо (горизонтально):**
```
Задача 1: Все tools
Задача 2: Весь граф
Задача 3: Весь prompt
```

**Хорошо (вертикально):**
```
Задача 1: Один tool + unit-тест
Задача 2: Граф с одним tool + mock LLM
Задача 3: Полный happy path диалог
Задача 4: Guardrails и edge cases
```

### 3. Шаблон задачи

```markdown
## Задача N: [краткое название]

**Описание:** один абзац.

**Критерии приёмки:**
- [ ] конкретное условие
- [ ] конкретное условие

**Верификация:**
- [ ] pytest tests/test_....py
- [ ] ручная проверка: ...

**Зависимости:** Задача M или «нет»

**Файлы:**
- path/to/file.py
```

## Правила

1. Задача ≤ ~100 строк нового кода или одна логическая единица
2. Каждая задача оставляет систему в рабочем состоянии
3. План на **русском**
4. После плана → `incremental-implementation`
